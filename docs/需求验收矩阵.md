# 需求验收矩阵

更新时间：2026-07-05

本文档用于把《需求分析文档_智创Agent》中的 F0-F7 需求映射到当前代码、页面、测试和烟测证据。状态口径：

- 已实现：当前仓库已有可运行接口或页面，并有测试或 smoke 证据。
- 已实现，需持续维护：首版可运行，但真实学校使用时需要继续维护资料或接入真实流程。
- 部分实现：已有演示闭环，但尚未覆盖需求完整边界。

## 当前剩余风险

1. 当前教学基础数据、学生画像、能力证据、学习计划、组队需求与状态、作业分析报告、任务中心、评测中心、知识库维护、本地账号目录和本地账号会话已接入 SQLite 能力，但正式学校使用仍需要接入统一身份认证。
2. RAG 首批资料已经满足数量和路径要求，但竞赛通知、课程大纲和 Rubric 需要按学校实际使用持续维护。
3. 作业代码分析当前支持 JSON 文件列表和 zip 压缩包上传，分析方式为轻量静态扫描；后续可接入仓库拉取、依赖安装隔离和更完整的测试运行沙箱。
4. 鸿蒙端当前为 P1 规划，主线仍是 Web 公网 Demo 和后续可迁移的 API 能力。
