# 首批知识库资料清单

本清单是首版系统需求，不绑定具体整理人。资料可由不同成员整理，最终统一进入知识库管理流程。

## 1. 首批重点路径

1. 算法竞赛。
2. AI 应用开发。
3. 软件项目实践。

## 2. 数量要求

| 类型 | 首版最低数量 | 用途 |
| --- | ---: | --- |
| 课程资料 | 5 门 | 支撑课程问答、项目分析、学习路径 |
| 竞赛资料 | 10 条 | 支撑竞赛推荐、训练计划、组队建议 |
| 项目案例 | 10 个 | 支撑软件实践、AI 应用开发、报告示例 |
| Rubric | 3 套以上 | 支撑代码项目评分和解释 |
| FAQ | 20 条以上 | 支撑常见问题回答 |

## 3. 课程资料建议

- 程序设计基础。
- 数据结构。
- 算法设计与分析。
- 软件工程。
- 数据库系统。
- Web 应用开发。
- 人工智能导论。

每门课程建议包含：

- 课程大纲。
- 章节知识点。
- 典型项目要求。
- 评分 Rubric。
- 推荐资料。

## 4. 竞赛资料建议

- 蓝桥杯。
- ACM/ICPC 训练资料。
- 中国大学生计算机设计大赛。
- 中国国际大学生创新大赛。
- 大学生服务外包创新创业大赛。
- 华为 ICT 大赛。
- 全国大学生信息安全竞赛。
- 互联网+ 校内训练资料。

每条竞赛资料建议包含：

- 竞赛简介。
- 适合学生类型。
- 能力要求。
- 准备周期。
- 推荐训练内容。
- 官方或校内资料来源。

## 5. 项目案例建议

- 课程项目管理系统。
- 校园二手交易平台。
- 智能问答知识库。
- RAG 文档问答系统。
- 代码评测与报告系统。
- 简历分析助手。
- 数据可视化看板。
- 小程序预约系统。
- AI 学习助手。
- 项目协作管理工具。

每个项目案例建议包含：

- 需求背景。
- 功能清单。
- 技术栈。
- 数据模型。
- 接口设计。
- 评价 Rubric。
- 可扩展方向。

## 6. 开源教学案例包资料

为适配开源创新大赛，首版知识库需要补充“八门课程交互式开源教学案例套件”。其中软件工程项目实践是 P0 详案，其他方向先以课程章节、课节、课堂演示、实验任务、系统支持能力和评价口径进入知识库。

建议至少包含：

| 类型 | 首版数量 | 用途 |
| --- | ---: | --- |
| 八门课程章节化大纲 | 1 套 | 对应开源创新大赛 8 个方向的 8 门课程、32 章、96 节 |
| 八方向课程案例矩阵 | 1 套 | 对应开源创新大赛 8 个方向的课程场景、动画演示和实验任务 |
| HTML 交互演示样例 | 1 套以上 | 支撑课堂投屏、演示视频和现场答辩 |
| 课程任务书 | 1 套 | 说明软件工程项目实践课程目标、阶段任务和交付要求 |
| 教师指南 | 1 份 | 说明教师如何使用项目学情看板完成课程讲评 |
| 学生指南 | 1 份 | 说明学生如何提交项目、查看报告和保存改进事项 |
| Rubric | 3 套以上 | 覆盖软件工程、数据库/Web 项目、AI 应用项目 |
| 项目报告样例 | 3 份以上 | 用于展示项目分析报告结构和证据口径 |
| 课堂运行记录 | 1 份以上 | 记录高仿真课堂或真实课堂的运行过程和效果指标 |

案例包建议目录：

```text
course-cases/
  open-source-interactive-teaching-suite/
    README.md
    course-syllabus.md
    direction-matrix.md
    classroom-script.md
    index.html
  software-engineering-project-practice/
    README.md
    syllabus.md
    task-book.md
    rubric.md
    teacher-guide.md
    student-guide.md
    sample-projects/
    knowledge-base/
    evaluation/
```

八门课程案例套件重点覆盖以下赛题方向：

- 深度学习框架与编程。
- 计算机系统导论。
- 编译原理。
- 数据库系统概论。
- 操作系统内核构建。
- 计算思维相关方向。
- 软件工程。
- AI 基础设施。

软件工程详案重点服务以下课程环节：

- 需求分析。
- 软件设计。
- 代码实现。
- 软件测试。
- 项目管理。
- 项目讲评与改进。
