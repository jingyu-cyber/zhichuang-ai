# 架构约定

## 1. 总体定位

智创Agent 是基于通用大模型的计算机学科垂类智能体应用，重点服务学校真实教学场景。系统以课程作业代码分析、教师学情诊断、学生成长路径、竞赛创新能力培养为核心。

## 2. 首版架构

```text
Web / HarmonyOS Client
        |
FastAPI API Gateway
        |
Service Layer ------------- SQLAlchemy / PostgreSQL
        |
AI Orchestration ---------- LangGraph
        |
Provider Adapter ---------- LLM / Embedding / Rerank
        |
RAG Pipeline -------------- ChromaDB(dev) / pgvector(prod)
```

## 3. 技术选型

| 层 | 开发期 | 部署期 |
| --- | --- | --- |
| Web 前端 | React + TypeScript + Vite + Ant Design + ECharts | 静态构建产物 |
| 鸿蒙端 | ArkTS + ArkUI | P1 单独建设 |
| 后端 | FastAPI + Pydantic | FastAPI 容器 |
| 数据库 | SQLite | PostgreSQL |
| ORM | SQLAlchemy + Alembic | SQLAlchemy + Alembic |
| 向量库 | ChromaDB | pgvector |

| 缓存/队列 | 可选 Redis | Redis |
| 部署 | 本地 / Docker Compose | 云服务器 / 校内服务器 |

## 4. 模块边界

### API 层

负责 HTTP 协议、请求校验、响应结构，不写复杂业务逻辑。

### Service 层

负责确定性业务逻辑：

- 权限判断。
- 课程、班级、作业、报告管理。
- 评分汇总。
- 数据库读写。
- 看板聚合。

### LangGraph 层

负责长任务、多阶段、多 Agent 协作：

- 代码作业分析。
- 学习计划生成和修改。
- 竞赛推荐。
- 组队推荐。
- 定期复盘。

### RAG 层

负责资料解析、切分、向量化、检索、引用校验和知识库评测。

### Provider Adapter 层

隔离模型厂商接口，业务代码只依赖统一接口：

- `LLMProvider`
- `EmbeddingProvider`
- `RerankProvider`

## 5. 状态和任务

长任务统一抽象为任务记录：

- `task_id`
- `task_type`
- `status`
- `input_snapshot`
- `graph_state`
- `result_ref`
- `error_message`
- `created_at`
- `updated_at`

状态枚举：

- `pending`
- `running`
- `waiting_user`
- `succeeded`
- `failed`
- `cancelled`

`waiting_user` 用于学生确认、补充目标、调整计划等用户主动参与场景，不用于教师审核作业报告。

## 6. 评分口径

能力画像和作业评分统一表达为“基于证据的相对画像”：

- 分数来自提交物、行为记录、Rubric 和知识库证据。
- 报告必须展示主要证据。
- 证据不足时应提示证据不足。
- 分数用于学习建议、教学诊断和推荐解释，不作为绝对能力判断。

## 7. 教师端定位

教师端是 P0 核心，不是扩展场景。

教师端展示：

- 班级提交情况。
- 作业维度分布。
- 学生个人作业报告。
- 班级共性问题。
- 讲评建议。

教师端不做：

- Agent 输出审核节点。
- 逐条校验学生能力证据的业务流程。
- 复杂教务系统替代。

## 8. 知识库范围

首批知识库聚焦：

- 算法竞赛。
- AI 应用开发。
- 软件项目实践。

资料类型：

- 课程大纲。
- 教材目录。
- 作业要求。
- 评分 Rubric。
- 竞赛资料。
- 项目案例。
- 代码样例。
- FAQ。

## 9. 迁移到鸿蒙的约定

Web 端不直接迁移为鸿蒙端。鸿蒙端 P1 使用 ArkTS + ArkUI 单独实现，复用后端 API、认证、任务状态和报告结构。

鸿蒙端优先做学生端高频链路：

- 登录。
- 个人画像。
- 学习路径。
- 任务提醒。
- 作业报告查看。
- 竞赛建议查看。

教师看板和知识库管理继续以 Web 为主。
