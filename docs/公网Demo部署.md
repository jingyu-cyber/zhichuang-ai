# 公网 Demo 部署说明

目标：部署一个可访问的智创Agent 演示环境，用于展示学生成长规划、项目分析、计划执行与阶段反馈、竞赛与组队推荐、知识库问答、教师看板、知识库管理和账号登录。

## 1. 演示入口

演示账号：

| 角色 | 账号 | 默认视图 |
| --- | --- | --- |
| 学生 | 林一舟 | 成长规划 |
| 教师 | 周老师 | 教师看板 |
| 管理员 | 平台管理员 | 知识库管理 |

演示主线：

1. 学生账号查看成长规划、能力画像、竞赛推荐和组队建议。
2. 查看自己的项目分析报告和下一步改进任务。
3. 在知识库问答中提问“如何准备算法竞赛？”。
4. 切换教师账号查看课程项目学情诊断和班级看板。
5. 切换管理员账号查看知识库资料清单和检索命中。

## 2. 本地 Docker Compose

准备环境：

```bash
make init
docker compose up --build
```

访问：

- 前端：`http://localhost:5173`
- 后端：`http://localhost:8000/api/health`

## 3. 服务器部署建议

建议配置：

- 2 核 CPU。
- 4GB 内存。
- 40GB 磁盘。
- Docker 和 Docker Compose。

部署流程：

```bash
git pull
cp .env.example .env
docker compose up -d --build
```

公网反向代理：

- 前端域名指向 `frontend:80`。
- 后端 API 域名或路径指向 `backend:8000`。
- 如果前端和后端不同域名，更新 `VITE_API_BASE_URL`。

学校身份接入：

- 在 `.env` 中设置 `SCHOOL_IDENTITY_SHARED_SECRET`，部署时不要使用示例值。
- 学校统一身份系统完成登录后，由受信任网关调用 `POST /api/auth/school-session`。
- 网关至少传入 `user_id`、`student_no`、`teacher_no` 或 `email` 中的一个字段，并在请求头携带 `X-School-Identity-Secret`。
- 平台只把身份断言映射到已导入的 SQLite 用户，课程、班级和本人访问范围仍由教学基础数据决定。

## 4. Smoke Check

```bash
./scripts/smoke.sh http://localhost:8000/api http://localhost:5173
```

检查项：

- 后端健康检查。
- 演示账号接口。
- 项目教师看板、教学改进建议和项目报告接口。
- 学生成长画像、学习计划、竞赛推荐、组队推荐。
- 教师竞赛梯队筛选和学生越权拦截。
- 计划执行事项保存与阶段反馈。
- 知识库资料新增、资料清单和检索接口。
- Agent 问答引用来源。
- 评测看板、课程列表。
- 本地学校账号会话和学校身份网关会话。
- 前端首页可访问。
